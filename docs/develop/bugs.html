bug记录规则

  1. id规则：id以#开头，使用时间戳，格式为YYYYMMDDHHMM，如2011年11月9号15时32分发现的bug，id为#201111091532
  2. 等级：
    高（严重错误，如内存溢出、页面假死、逻辑错误、功能无法使用等）；
    中（一般错误，如兼容性问题、数据不完整、文字错误、响应较慢等，不影响功能使用）；
    低（优化建议，对系统的优化改进建议）
  3. 环境：操作系统、浏览器(明确版本)、其他环境说明
  4. 描述：对bug的产生场景、造成的影响进行描述。
  5. 完整记录：id+（等级）：[环境]描述
  6. 处理：已修正的bug使用删除线标记，且在bug下方记录解决方案。

bug记录

#201201091640(低)：[所有浏览器-from niejing]建议对折叠面板组控件增加打开方式的配置，共有两种方式，一种是只能同时打开一个面板，一种是随意打开或关闭任意面板。

#201201091630(中)：[所有浏览器-from niejing]删除导航栏控件中的某个按钮后，按钮组宽度无法自动补齐，显示怪异。

解决方案：对导航栏的删除单独处理，删除某个导航按钮后重新初始化控件。

#201112221426(高)：[所有浏览器-from wangyi]
在工作区页面空白处点击，然后选择删除操作，删除了内容区，之后无法向工作区拖拽控件。

解决方案：内容区为拖拽容器，禁止删除。修改删除逻辑，在选中内容区后不显示删除操作，禁止删除动作。

#201112211640(低)：[所有浏览器-from jinye]建议将控件从工作区拖出的时候也能删除。

#201112211636(低)：[所有浏览器-from jinye]创建app主界面主内容区的高度不能完美自适应。

解决方案：使用绝对定位和js计算，已解决该问题。

#201111091544(低)：[windows7、所有浏览器]页面文字无法选择和复制。

#201110191423(低)：[windows7、safari5、chrome12]
在长页面上使用鼠标滚轮滚动，滚动很慢且不灵敏。

解决方案：由iscroll.js改变浏览器默认行为导致，修改iscroll.js中的wheelDeltaY = e.wheelDeltaY / 12为wheelDeltaY = e.wheelDeltaY / 1。
