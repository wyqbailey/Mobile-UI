媒体查询助手 Media Query Helper Classes

注意: 这个特性还是beta版，前途未知，推荐使用CSS3的媒体查询.

jQuery Mobile 添加一个classes到HTML中来模拟浏览器的方向类似于 min/max-width CSS media queries.这个classes会在加载、重置大小和方向改变时更新，允许你在css中截断这些classes,以创建 快速布局 - 即使浏览器不支持媒体查询功能!

方向类 Orientation Classes

HTML元素总包含 "portrait" or "landscape"两个类，其值取决于设备是横向还是竖向.你可以向下面一样在css里定义它们:

			
.portrait {
	/* 纵向的变化定义! */
}
.landscape {
	/* 纵向的变化定义! */
}

最小/最大宽度断点类 Min/Max Width Breakpoint Classes

默认情况下，我们提供如下宽度断点： 320,480,768,1024. 对应的class: "min-width-320px", "max-width-480px", 意味着它们可以应用在 替换(或附加) 模拟媒体查询值.

			
.myelement { 
	float: none;
}			
.min-width-480px .myelement {
	float: left;
}

Jquery Mobile中的许多插件都利用了这些宽度折断点，举个例子：当浏览器宽度在480px以上时，表单元素会浮动在label的旁边. 约束表单文本框的CSS在定义这个行为时是这样 :

			
label.ui-input-text { 
	display: block; 
}
.min-width-480px label.ui-input-text { 
	display: inline-block; 
}

新增宽度断点 Adding Width Breakpoints

利用自定义的宽度断点, jQuery Mobile 提供了 $.mobile.addResolutionBreakpoints 方法, 它接受一个单一的数字或数组，当使用到他们时，就会增加一个最小/最大的宽度断点。

			
//add a min/max class for 1200 pixel widths			
$.mobile.addResolutionBreakpoints(1200);

//add min/max classes for 1200, and 1440 pixel widths			
$.mobile.addResolutionBreakpoints([1200, 1440]);

运行媒体查询 Running Media Queries

jQuery Mobile提供了一个称为方法，允许你检测是否支持媒体查询. 只需要调用 $.mobile.media()，然后传递一些参数（media type or query）. 如果此媒体支持，则返回true，否则为false

			
//test for screen media type
$.mobile.media("screen");

//test  a min-width media query
$.mobile.media("screen and (min-width: 480px)");

//test for iOS retina display
$.mobile.media("screen and (-webkit-min-device-pixel-ratio: 2)");
